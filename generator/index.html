ジェネレーター

概要

たとえば、range(0, 1000000)は100MBを超えるメモリを消費することになる。ジェネレーターは、普遍的にはこのような「巨大または要素数無限の配列を定義し、ひとつひとつ処理をしなければならない場合」にその回避策として用意されている。わかりやすいかどうか、「（広義の）ストリーム処理」の一種を実現するものといった感じ。

基本

実際にrange(0, 1000000)のループをするのも気持ちが悪いので、今回はrange(1, 3)の以下の処理と結果的に同じものをGeneratorで実現してみる。

foreach (range(1, 3) as $num) {
		echo "$num ";
}
// 結果: 1 2 3

Generatorオブジェクトは、一つ以上のyield命令を含む関数の返り値として次のように定義する。

function gen_int(int $max): Generator {
    for ($i = 1; $i <= $max; $i++) {
        yield $i;
    }
}

$generator = gen_int(3);

Generatorクラスは、Iteratorインターフェースを実装しているため、$generatorに対してnext(), current()などのメソッドが使える。yield文で処理を一時停止し、next()が呼ばれることをトリガーに、次のyield文まで処理を進めて値を返す。という流れになる。

echo $generator->current() . " "; // 1
$generator->next();
echo $generator->current() . " "; // 2
$generator->next();
echo $generator->current() . " "; // 3

つまり、2がメモリ上に確保されているときには、1は破棄済み、3は未定義でほかの値はメモリを占有していない。このようにストリーム処理が実現される。「ジェネレーター」と言う意図は都度「生成するもの」ということだろう。

実際は、このように手動でnext()やcurrent()を呼ぶのは稀有。ジェネレーターの目的は配列のように、管理する各値を使って処理をすることなので、Generatorオブジェクトもちゃんと配列ライクにforeachに渡せるように設計されている。foreach文では、このようなnext()やcurrent()は構文のうちに隠蔽される。

foreach (gen_int(3) as $num) {
		echo "$num ";
}
// 結果: 1 2 3

他の用途

ChatGPTによれば、他にもジェネレーターは以下のような用途で使用されるらしい。（未検証）

  • 無限シーケンスの生成

    終端がないデータ列（例：自然数列、乱数列、時刻ストリームなど）を扱うときに、配列ではそもそも実現できません。

    // 1 から順にカウントし続ける無限シーケンサー
function infiniteCounter(): Generator {
    $i = 1;
    while (true) {
        yield $i++;
    }
}
foreach (infiniteCounter() as $n) {
    echo $n, "\n";
    if ($n >= 10) break;  // 必要なところで止める
}
  • コルーチン（双方向通信）

    Generator は send() や throw() を使って、外部から値を受け取ったり例外を注入したりできます。これにより、簡易的な 協調的マルチタスク や パイプライン処理 を実装可能です。

    /**
 * ロガーコルーチンの例
 * send() でメッセージを受け取り、内部で処理する
 */
function logger(): Generator {
    while (true) {
        // 外部から送られてくる文字列を受け取る
        $msg = yield;
        // 受け取ったときにのみログ出力
        echo "[LOG] ", $msg, "\n";
    }
}

$log = logger();
$log->current();          // ジェネレーターを初期化（最初の yield まで進める）
$log->send('開始します'); // ⇒ [LOG] 開始します
$log->send('処理中…');    // ⇒ [LOG] 処理中…
  • ストリーミングフィルタ／パイプライン

    大量ストリームデータ（ファイル行やソケット受信バッファなど）を、複数のフィルタ処理で順次変換しつつ受け渡すときにも有効です。

    // ファイルの行を一行ずつ yield
function readLines(string $path): Generator {
    $fh = fopen($path, 'r');
    while (($line = fgets($fh)) !== false) {
        yield $line;
    }
    fclose($fh);
}

// 文字列フィルタの例（大文字化）
function toUpperCase(Generator $src): Generator {
    foreach ($src as $line) {
        yield strtoupper($line);
    }
}

// パイプラインで結合
foreach (toUpperCase(readLines('data.txt')) as $uLine) {
    echo $uLine;  // ファイルを一行ずつ大文字で出力
}
